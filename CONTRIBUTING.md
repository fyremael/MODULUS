# Contributing

## Contribution Requirements

1. You must have the legal right to submit all content you contribute.
2. You must not submit code copied from sources with incompatible license terms.
3. You must disclose AI-assisted contributions in the pull request description.
4. You must include origin references for any substantial imported logic.

## Developer Certificate of Origin (DCO)

By submitting a contribution, you certify:

`Developer Certificate of Origin 1.1`

`(a) The contribution was created in whole or in part by me and I have the right to submit it under the open source license indicated in the file; or`

`(b) The contribution is based upon previous work that, to the best of my knowledge, is covered under an appropriate open source license and I have the right under that license to submit that work with modifications, whether created in whole or in part by me, under the same open source license; or`

`(c) The contribution was provided directly to me by some other person who certified (a), (b), or (c) and I have not modified it.`

`(d) I understand and agree that this project and the contribution are public and that a record of the contribution (including personal information I submit with it) is maintained indefinitely and may be redistributed consistent with this project or the open source license(s) involved.`

## Prohibited Sources

- Proprietary code without explicit written permission.
- Licensed material requiring source disclosure obligations not approved for this project.
- Content with unknown provenance.

## AI-Assisted Code Disclosure

When AI tools are used, include:

1. Tool and version (example: `ChatGPT 5.2`).
2. High-level prompt intent (no confidential data).
3. Human reviewer responsible for technical and legal checks.
4. Confirmation that output complies with project licensing policy.
