# Security Policy

## Supported Scope

This repository is under active diligence and stabilization. Security reports are accepted for all current files.

## Reporting a Vulnerability

1. Do not open a public issue for unpatched vulnerabilities.
2. Report privately to the project owner with:
   - affected file/module
   - impact summary
   - reproduction steps
   - suggested fix (if available)
3. Expect acknowledgment within 5 business days.

## Disclosure Process

1. Triage and severity assessment.
2. Patch development and validation.
3. Coordinated disclosure after fix availability.
